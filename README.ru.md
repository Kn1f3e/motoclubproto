# VSE

Сайт-проект для популяризации мототуризма.

## Технологический стек

- Vue 3
- Vite
- Vue Router
- JavaScript (ES modules)
- CSS3

## Возможности

- Страница аутентификации (регистрация / вход / восстановление пароля)
- Главная страница с:
  - левым сворачиваемым меню разделов
  - главным блоком
  - автопрокручиваемым фото-слайдером
  - плавной анимацией как для базовых, так и для динамически добавленных/удаленных разделов
- Раздел `О клубе` с подстраницами:
  - Устав клуба
  - Друзья и партнеры
  - Фото и видео из жизни клуба
  - Планы и развитие клуба
- Страница маршрутов
- Страница ближайших событий
- Страница `МотоМаркет и Сервис`
- Страница профиля с вкладками:
  - Профиль
  - Нашивки
  - Настройки
  - Уведомления
- Панель администратора с 3 рабочими блоками:
  - Выдача нашивок (выдать/снять)
  - Пользователи (смена роли)
  - Контент сайта (главный блок + разделы)
- Управление контентом из админки:
  - редактирование существующих разделов
  - добавление нового раздела через выбор "Добавить новый"
  - удаление и сброс разделов к стоковым
  - редактирование карточек раздела (оглавление/подглава/текст)
- Динамические пользовательские маршруты разделов (например `/test`) на основе данных админки
- Стилизованные модальные подтверждения для опасных действий (вместо системного confirm)
- Страница 404
- Настройки темы и языка в профиле
- Плавные анимации появления страниц и карточек
- Кнопка прокрутки страницы вверх
- Локальное сохранение состояния (тема/язык/данные профиля)
- Сохранение админ-сессии в local storage

## Требования

- Node.js 18+
- npm 9+

## Запуск проекта локально

1. Установите зависимости:

```bash
npm install --include=dev
```

2. Запустите dev-сервер:

```bash
npm run dev
```

3. Откройте адрес из терминала (обычно `http://localhost:5173`).

## Сборка для production

```bash
npm run build
```

Результат сборки будет в папке `dist/`.

## Структура проекта

- `src/pages/` — страницы приложения (`AuthPage`, `HomePage`, `ProfilePage`, подстраницы клуба и т.д.)
- `src/router/index.js` — маршруты приложения
- `src/state/AppStore.js` — состояние приложения (пользователь, тема, язык, UI-состояние)
- `src/styles/index.css` — точка подключения стилей
- `src/styles/core/` — базовые глобальные стили
- `src/styles/components/` — компонентные стили (`header.css`, `toasts.css`, `profile-sidebar.css` и др.)
- `src/styles/pages/` — стили страниц (`auth.css`, `home.css`, `detail.css`, `profile.css`)
- `src/styles/responsive/` — адаптивные стили
- `src/assets/` — локальные ассеты (`flag.png`, `logowithoutback.png`)
- `public/` — публичные статические файлы (`ustav.docx`)

## Карта CSS (по файлам)

- `src/styles/core/global.css` — токены темы, базовые теги, общие анимации и базовые блоки.
- `src/styles/components/layout-shell.css` — каркас приложения и сервисные UI-элементы.
- `src/styles/components/toasts.css` — всплывающие уведомления.
- `src/styles/components/header.css` — хедер и адаптив хедера.
- `src/styles/components/profile-sidebar.css` — боковое меню профиля и мобильное выдвижение.
- `src/styles/pages/auth.css` — стили страницы авторизации.
- `src/styles/pages/home.css` — стили главной (меню разделов, hero, слайдер).
- `src/styles/pages/detail.css` — стили страниц разделов/подразделов.
- `src/styles/pages/profile.css` — центральный контент профиля.
- `src/styles/responsive/tablet.css` — адаптив до `900px`.
- `src/styles/responsive/mobile.css` — мобильные эффекты валидации.

## Known Limitations

- На главной в слайдере сейчас используется одно локальное изображение (`src/assets/flag.png`) для всех слайдов.
- В проекте пока нет backend/API; состояние профиля и авторизации хранится локально.
- Админ-функции сейчас работают в локальном demo-режиме (без серверного RBAC и БД).
- Часть текстового контента и контактных данных сейчас является демонстрационной.
